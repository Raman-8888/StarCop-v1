                      STARCOP - PROJECT SUMMARY
            A Next-Gen Startup & Investor Connection Platform

WHAT IS THIS PROJECT?
---------------------
This is a modern, high-end platform designed to bridge the gap between Startups
and Investors. It's not just a directory; it's a "Digital Soul" ecosystem where
vision meets capital.

Think of it like a professional dating app for businesses, but with stunning 
3D graphics, real-time chat, and a sophisticated matching system. Startups 
showcase their vision, and Investors find their next unicorn.

                        CHAPTER 1: FRONTEND
                    (The Part You See and Click)

WHAT IS THE FRONTEND?
--------------------
The frontend is the beautiful interface you interact with. It uses advanced 3D 
graphics (via Three.js) to create an immersive "Universe" feel, combined with 
smooth, professional dashboards for managing deals and connections.


MAIN TECHNOLOGIES USED:
----------------------
1. React 19 - The core framework for the user interface
2. Three.js (@react-three/fiber) - Powers the 3D Hero scenes and effects
3. TailwindCSS v4 - Provides the sleek, modern styling
4. Framer Motion - Handles all the smooth animations and transitions
5. Socket.io Client - Enables real-time chat and notifications

WHAT IS THE BACKEND?
-------------------
The backend is the heavy lifter. It runs on Node.js and acts as the secure vault 
for all data. It validates every request, manages the database, handles 
real-time socket connections, and integrates with AI services.


MAIN TECHNOLOGIES USED:
----------------------
1. Express.js - The web server framework
2. MongoDB (Mongoose) - The NoSQL database for flexible data schemas
3. Socket.io - The real-time engine for chat
4. JWT (JSON Web Tokens) - The security pass for authentication
5. Cloudinary & Supabase - Cloud storage for images and documents
6. Google Gemini AI - For intelligent feature integration

HOW THE BACKEND WORKS (SIMPLE EXPLANATION):
------------------------------------------

1. REGISTRATION
   â†’ Startup signs up
   â†’ Backend validates data
   â†’ Hashes password with Bcrypt
   â†’ Saves to MongoDB 'users' collection

2. REAL-TIME CHAT
   â†’ User A sends message
   â†’ Backend saves to MongoDB
   â†’ Socket.io immediately "pushes" message to User B's screen
   â†’ No page refresh needed!

3. MATCHMAKING
   â†’ Investor searches for "FinTech"
   â†’ Backend queries 'users' collection where industry="FinTech"
   â†’ Returns matching profiles

4. FILE UPLOADS (Pitch Decks / Avatars)
   â†’ User selects file
   â†’ Middleware (Multer) intercepts request
   â†’ Controller uploads to Cloudinary/Supabase
   â†’ Secure URL is saved in Database

5. SECURITY
   â†’ Every API request checks for a valid "Bearer Token"
   â†’ If token is fake or expired, Backend says "401 Unauthorized"
   â†’ Protects private data from hackers


================================================================================
                        HOW EVERYTHING CONNECTS
================================================================================

THE COMPLETE FLOW:
-----------------

1. USER LANDS ON SITE
   - Frontend loads 3D Hero Scene
   - "Digital Soul" animation plays

2. USER LOGS IN
   - Frontend sends credentials to Backend
   - Backend verifies and sends back a Token
   - Frontend saves Token in memory/storage

3. USER BROWSES OPPORTUNITIES
   - Frontend calls `GET /api/opportunities`
   - Backend fetches list from Database
   - Frontend renders them as "Cards"

4. USER STARTS CHAT
   - Frontend emits `join_chat` socket event
   - Backend connects the two users in a virtual "Room"
   - They exchange messages instantly

5. USER UPLOADS PITCH DECK
   - Frontend sends file via FormData
   - Backend (Multer) receives file
   - Uploads it to Cloud (Supabase/Cloudinary)
   - Saves the Link in the Database
